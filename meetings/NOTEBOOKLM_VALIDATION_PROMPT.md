# NotebookLM 회의 결정 검증 질문

NotebookLM 소스에 다음 두 문서를 넣는다.

1. `Knowledge/Meetings/meeting_decision_criteria.md`
2. 검증할 `meetings/summary/YYYY-MM-DD_meeting.md`

아래 질문을 그대로 사용한다.

```text
KIBA 회의 의사결정 기준 v1만을 기준으로 이번 회의록의 '결정 사항'과 '할 일'을 검증해 주세요.

1. 관련 기준 ID별로 통과/조건부/보류/해당 없음 중 하나를 판정하세요.
2. 각 판정에 회의록의 근거 문장을 붙이세요. 근거가 없으면 '근거 없음'이라고 쓰세요.
3. RSK 기준의 보류 항목, 담당자·기한·이슈 연결이 빠진 할 일을 먼저 표시하세요.
4. 실행 전에 반드시 보완할 항목을 최대 5개로 정리하세요.
5. 회의록에 없는 사실은 추측하지 마세요.

출력 표: 기준 ID | 판정 | 근거 | 보완 조치 | 담당자/기한
마지막 줄: 최종 결론 = 실행 / 조건부 실행 / 보류
```

검증 결과의 핵심은 회의록 `## 판단 기준 검증`에 옮기고, 보완 조치는 `## 할 일`에 추가한다.
