## git remote 잘못된 저장소로 push 정정
**도구:** Claude
**질문:** `git remote add origin`이 "already exists"로 실패하고, push가 의도한 `kiba_2026`가 아니라 옛 `kiba.git`로 들어갔다.
**응답 요약:**
- 원인: origin이 이미 옛 주소를 가리켜 `add`가 무시됨 → `git remote set-url origin <새 주소>`로 교체해야 함.
- 교체 후 `kiba_2026`로 정상 push 확인. 39MB 용량의 주범은 `docs/`의 40MB hwp 파일로 확인.
