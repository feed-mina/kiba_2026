# KIBA 사이트 사용 방법 및 배포 복구 보고서

## 1. 보고 목적
- KIBA 정적 사이트가 GitHub Pages에서 404로 노출되는 원인을 복구하고,
- 원장님과 관계자가 바로 사용할 수 있도록 사이트의 목적과 이용 절차를 정리한다.

## 2. 문제 원인
- 기존 저장소에는 정적 페이지 배포용 GitHub Actions 워크플로가 없어 GitHub Pages가 자동으로 갱신되지 않았다.
- 페이지 진입 경로가 비공개 상태이거나 배포가 미반영되면 URL 접속 시 404가 발생할 수 있다.

## 3. 해결 내용
- GitHub Pages 자동 배포 워크플로를 추가했다.
- 홈페이지 진입용 404 페이지를 추가해 잘못된 경로에서도 홈으로 복귀할 수 있게 했다.
- 원장님 보고용 보고서를 PDF로도 저장해 배포 자료로 사용할 수 있게 했다.

## 4. 사이트 사용 방법
1. 브라우저에서 GitHub Pages 주소로 접속한다.
2. 상단의 "프로젝트 진행 상황 보기" 섹션에서 주차별 진행 상황을 확인한다.
3. 지식베이스, 원가계산서, 이슈 보드 링크를 통해 세부 자료로 이동한다.
4. 문제가 생기면 배포가 완료된 뒤 다시 접속하면 정상 확인 가능하다.

## 5. 운영 권장사항
- main 브랜치에 변경이 반영되면 GitHub Actions가 자동으로 Pages를 재배포한다.
- 배포 상태는 Actions 탭에서 확인한다.
- 필요 시 Pages 설정에서 "GitHub Actions"를 소스로 선택한다.
