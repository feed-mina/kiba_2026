# 협회 등록 요건 판정 엔진 — 설계 (2026-06-19)

> 관련 이슈: #1 / #11 (엔지니어링협회 등록 검토 — 본 엔진의 **샘플 1**)
> 목적: "엔지니어링협회 등록"을 고정 기능이 아니라, **여러 협회·직업인 유형에 재사용 가능한 "등록 요건 판정 엔진"** 으로 일반화한다.
> ⚠️ 내부 설계 문서. 직원 PII는 비공개 `docs/`에서만 다루고, 공개 이슈/페이지에는 집계만 올린다.

---

## 1. 배경

- 지금은 엔지니어링협회(건설사업관리) 등록을 위한 1회성 검토로 진행 중이나, **건설기술인협회·소프트웨어기술자·감정평가 등 협회 등록 일이 계속 늘어날 전망**.
- 매번 협회를 코드/문서에 하드코딩하지 않고, **협회명·직업인 유형·요건·법적 근거를 모두 변수(데이터)** 로 두어야 한다.
- 핵심은 **"협회에 어떤 직업인으로 등록하느냐에 따른 세부 요건(법적 근거·협회 기준)을 찾아 데이터로 뽑는 것"**.

---

## 2. 핵심 원칙

Codex가 제시한 방향(고정 기능 → 일반화 엔진)을 채택하되, 2026-06-19 실제 작업에서 드러난 문제(ICEC 산식, ID 불일치, 키워드 오분류, 공개/비공개)를 반영해 다음을 강화한다.

### 2-1. 변수화 매핑
| 고정값(현재) | 변수화(목표) |
|---|---|
| 엔지니어링협회 | `협회명` |
| 건설기술인/엔지니어링기술자 | `등록 직업인 유형` |
| 등록 분야(건설사업관리) | `등록 분야` |
| 특급 1명 + 초급 이상 10명 | `요건 규칙 데이터` |
| 법령/별표/협회 기준 PDF | `근거 문서 데이터` |

### 2-2. 가장 중요한 구조적 개선 — 2층 분리
Codex 안은 "등급 산정"과 "등록 요건"을 한 테이블에 섞었으나, 둘은 성격이 다르므로 분리한다.

- **개인 등급 산정 모델(person-level)**: 사람 1명의 등급을 계산.
  - 건설기술인 = ICEC 역량지수(`자격+학력+경력+교육-행정처분`) → 별표3
  - 엔지니어링기술자 = 다른 산식, 소프트웨어기술자 = 또 다름
  - → **직업인 유형별 "등급 산정 모듈"을 플러그인처럼** 꽂는다. (오늘 만든 ICEC 계산기 = 첫 모듈)
- **조직 등록 요건 규칙(org-level)**: 협회/분야가 요구하는 **분포**(특급1+초급10, 자본금 1.5억 등).

> 핵심 함의: **같은 직원이 협회마다 다른 등급**이 된다(건설기술인 vs 엔지니어링기술자 경력 계산이 다름). 그래서 산정 모델과 요건 규칙을 분리해야 재사용된다.

## 4. 데이터 모델 (스키마 초안)

### 4-1. 협회 등록 프로그램 (`registration_program`)
| 필드 | 설명 |
|---|---|
| program_id | 키 (예: `KISCON-CM`, `ENG-PM`) |
| 협회명 | 한국건설기술인협회, 한국엔지니어링협회 … |
| 등록 직업인 유형 | 건설기술인 / 엔지니어링기술자 / 소프트웨어기술자 … |
| 등록 분야 | 건설사업관리 / 설계·시공 / 품질관리 … |
| 등급 산정 모듈 | `icec_v2024` 등 (4-3 참조) |
| 적용 법령·근거 | 근거 문서 ID 목록 |
| 시행일·버전 | 규칙셋 버전 (개정 추적용) |

### 4-2. 등록 요건 규칙 (`requirement_rule`) — 버전·근거 필수
```json
{
  "rule_id": "KISCON-CM-2021.09.14-인력",
  "program_id": "KISCON-CM",
  "분야": "건설사업관리",
  "항목": "최소인원",
  "조건": { "총원": 10, "최소등급": "초급", "특수": [{"등급":"특급","최소":1}] },
  "기타요건": [ {"항목":"자본금","값":"150000000","단위":"원"}, {"항목":"사무실"} ],
  "근거": {
    "법령": "건설기술 진흥법 시행령 별표5",
    "조문": "제44조제2항",
    "시행일": "2021-09-14",
    "url": "https://www.law.go.kr/법령/건설기술진흥법시행령",
    "원문발췌": "건설사업관리: 특급기술인 1명을 포함한 초급 이상의 건설기술인 10명 이상",
    "비고": "비고1: 기술인력=법 제21조제1항 신고(등록)자"
  },
  "상태": "확정",          // 초안 | 사람검토 | 확정 — 확정만 판정에 사용
  "검토자": "", "검토일": ""
}
```

### 4-3. 등급 산정 모듈 (`grade_model`) — 직업인 유형별
예) 건설기술인 ICEC (별표3):
```
역량지수 = 자격지수(40) + 학력지수(20) + 경력지수(40) + 교육지수(5) - 행정처분(3)
 자격지수: 기술사/건축사40, 기사/기능장/산업안전지도사30, 산업기사20, 기능사15, 기타10
 학력지수: 학사+20, 전문3년19/2년18, 고졸15, 지정교육12, 비전공10 (전문분야 석사+1.5/박사+3)
 경력지수: (logN/log40)×100×0.4, N=인정경력일/365×책임정도 보정계수
 교육지수: 협회 교육 35h당 1점(역량강화 2점), 이수 후 3년
등급(건설사업관리): 초급40 / 중급60 / 고급70 / 특급80
※ 별표3 비고3: 자격·학력 직무분야 ≠ 실제 수행분야면 역량지수 40+라도 초급 고정·승급 불가
```

### 4-4. 직무분야 인정 종목 매핑 (`cert_field_map`) — 키워드 금지
- 별표1·별표4(자격종목별 전문분야)를 **데이터 테이블**로. (자격증명 키워드로 가르면 품질관리·화공 기술사 오분류 발생)
- 필드: 자격종목, 직무분야, 전문분야, 자격지수배점.

### 4-5. 직원 판정 결과 (`eligibility_result`)
```json
{
  "직원": "(비식별키)", "program_id": "KISCON-CM",
  "역량지수": 80.5, "등급": "특급",
  "구성": {
    "자격지수": {"값":40, "출처":"quali-fit", "신뢰도":"확정"},
    "학력지수": {"값":10, "출처":"quali-fit", "신뢰도":"확정"},
    "경력지수": {"값":30.5, "출처":"취득일 프록시", "신뢰도":"추정"},
    "교육지수": {"값":0, "출처":"-", "신뢰도":"미확보"}
  },
  "판정": "조건부 가능",
  "부족·필요": ["협회 경력증명(인정경력)", "건설기술인 신고/등급 확정", "최초교육 이수"],
  "근거": ["KISCON-CM-2021.09.14-인력"]

```
> **각 입력값에 `출처·신뢰도(확정/추정/미확보)`** 를 1급 필드로. → "조건부 가능"이 곧 **수집 체크리스트**가 된다.

## 5. 판정 출력 (단순 가능/불가를 넘어서)

1. **조직 충족 시뮬레이션**: "특급1+초급10"을 만족하는 최소 인원 조합을 풀고 갭 표시.
   - 예) "건설기술인협회 신고 8명 → **2명 추가 신고 필요**; 후보 A·B는 경력증명만 받으면 초급 확정"
2. **등급 상향 최적화**: 사람별 *가장 싼 승급 경로* (예: "경력지수 +X면 특급", "기사 취득 시 +30").
3. **다음 할 일** 리스트(증빙·신고·교육) 자동 생성.

## 6. 재사용 자산 (새로 만들지 말 것)

- **quali-fit**: `scoring.py`·`validation.py`·`db.py` + 직원/학력/자격 DB(`Data/*.csv`) 이미 존재 → **여기에 "협회 등록 적격 판정" 모듈 추가**가 신규 시스템보다 유리.
- **ICEC 계산기**(2026-06-19 구현) = 첫 등급 산정 모듈.
- **협회 등록현황 로스터**(`docs/협회등록현황_2026-06-19.xlsx`) = "현재 등록상태(기존/신규)" 데이터 소스. 건설기술인협회 기존3+신규5=8명.
- **근거 문서**: `docs/[별표 3]…pdf`(등급산정), `docs/[별표 5]…pdf`(등록요건), `학력_자격증_26.05.14_v13.xlsm`.

## 7. 프라이버시·거버넌스·품질 (빌트인)

- **출력 모드 2종 내장**: 비식별 집계(공개 이슈) / 식별 상세(비공개 `docs/`). 매번 손으로 가르지 않도록.
- **사람 확정 게이트**: 요건 규칙은 `상태=확정`만 판정에 사용. **AI는 "요건 후보"까지만**, 확정은 사람.
- **데이터 품질 게이트**(2026-06-19 실제 발생 이슈 반영):
  - 직원 식별 키 정규화 — Codex xlsx ID(`EMP0001`) ≠ quali-fit CSV(`EMP-001`) → **이름 등 정규화 키**로 결합.
  - 날짜 형식 정규화 — `YY.MM.DD` / `YYYY-MM-DD` 혼재.
  - 자격 분류는 키워드 금지 → **직무분야 인정 종목 매핑표** 사용.
- **감사 추적**: 모든 판정은 근거(법령 조문·시행일·URL·원문발췌·해시)로 역추적 가능해야 협회 제출/감사에 견딘다.

## 8. 추진 단계 (MVP 우선)

1. **샘플 1 완성**: 엔지니어링/건설사업관리 1개를 끝까지 — 요건 규칙 + ICEC 모듈 + 비식별/식별 출력. (= #11)
2. **두 번째 협회 추가**(예: 소프트웨어기술자)로 추상화 검증 ← *일반화의 진짜 시험대*.
3. PDF/엑셀 보고 자동 생성(원장 보고용).
