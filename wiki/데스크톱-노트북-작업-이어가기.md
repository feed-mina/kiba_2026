# 데스크톱-노트북 작업 이어가기

목표: 낮에는 데스크톱, 밤에는 노트북에서 같은 KIBA 프로젝트 상태를 이어서 작업한다.

## 기준

- 코드/ASK/Todo/Knowledge/wiki: GitHub `feed-mina/kiba_2026`
- 비공개 원문 문서: Cloudflare R2 `kiba-docs-private`
- 로컬 실행 자동화: 각 PC의 Windows 작업 스케줄러
- Obsidian: 각 PC에서 `C:\Users\User\Desktop\KIBA` 같은 로컬 repo 폴더를 vault로 열기

## 새 PC 세팅 순서

1. GitHub에서 `feed-mina/kiba_2026`를 clone 또는 pull한다.
2. Python, Git, GitHub CLI, rclone을 설치한다.
3. R2 동기화 자격증명을 만든다.
   ```powershell
   .\scripts\setup_r2_sync.ps1
   ```
4. 문서 다운로드 비밀번호를 현재 Windows 사용자 기준으로 저장하고 스케줄을 등록한다.
   ```powershell
   .\scripts\setup_docs_schedule.ps1
   ```
5. Pages 버튼으로 로컬 수동 실행할 수 있게 URL 프로토콜을 등록한다.
   ```powershell
   .\scripts\setup_local_run_protocol.ps1
   ```
6. Obsidian에서 repo 폴더를 vault로 연다.

## 매일 작업 전후

- 시작 전:
  ```powershell
  git pull --rebase origin main
  .\scripts\download_docs_scheduled.ps1 -SkipDownload
  ```
- 종료 전:
  ```powershell
  .\scripts\download_docs_scheduled.ps1 -SkipDownload
  git status --short
  ```

## 주의

- `scripts/.docs_password.xml`, `scripts/.r2_credentials.xml`은 PC/Windows 사용자별 DPAPI 파일이라 복사해도 다른 PC에서 풀리지 않는다.
- `docs/` 원문은 Git에 올리지 않고 R2와 로컬 동기화로 맞춘다.
- GitHub Wiki는 `Knowledge/`에서 생성한 `wiki/`를 `scripts/publish_github_wiki.ps1`로 별도 push한다.
